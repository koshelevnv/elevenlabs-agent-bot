# Узлы воркфлоу — ElevenLabs Bot

## Входная точка

| Название | Назначение |
|---|---|
| **ТГ: Входящие** | Принимает все входящие сообщения и нажатия кнопок от пользователей Telegram |

---

## Главный роутер

| Название | Назначение |
|---|---|
| **Роутер: Тип сообщения** | Определяет тип входящего события: `/start`, `/agents`, нажатие кнопки (callback) или произвольный текст |

---

## Ветка /start

| Название | Назначение |
|---|---|
| **БД: Регистрация** | UPSERT пользователя в БД. Возвращает `elevenlabs_api_key` для проверки |
| **Проверка: Ключ при старте** | Проверяет наличие ElevenLabs API ключа у пользователя |
| **ТГ: Приветствие** | Отправляет приветственное сообщение (если ключ уже есть) |
| **БД: Запросить ключ** | Устанавливает `pending_action = 'setkey'` — бот ждёт ключ от пользователя |
| **ТГ: Нет ключа** | Просит пользователя прислать ElevenLabs API ключ |

---

## Ветка /agents

| Название | Назначение |
|---|---|
| **БД: Авторег /agents** | UPSERT пользователя (на случай если не было /start) |
| **БД: Получить ключ /agents** | Загружает `elevenlabs_api_key` текущего пользователя |
| **Проверка: Есть ключ** | Проверяет наличие API ключа. Если нет — запрашивает |
| **БД: Запросить ключ** | Устанавливает `pending_action = 'setkey'` (общий нод с веткой /start) |
| **ТГ: Нет ключа** | Просит прислать ElevenLabs API ключ (общий нод с веткой /start) |
| **EL: Получить агентов /agents** | GET-запрос к ElevenLabs API — получает актуальный список агентов пользователя |
| **Код: Синх агентов /agents** | Формирует SQL для синхронизации: DELETE устаревших + UPSERT актуальных агентов |
| **БД: Удалить старых агентов /agents** | Удаляет из БД агентов которых больше нет в ElevenLabs |
| **БД: Upsert агентов /agents** | Добавляет/обновляет актуальных агентов в БД |
| **БД: Получить агентов** | Загружает список агентов пользователя из БД |
| **Код: Клавиатура агентов** | Формирует inline-клавиатуру со списком агентов. Если агентов нет — показывает соответствующее сообщение |
| **ТГ: Список агентов** | Отправляет список агентов с кнопками в Telegram |

---

## Ветка нажатия кнопок (callback)

| Название | Назначение |
|---|---|
| **Роутер: Тип callback** | Определяет тип нажатой кнопки: выбор агента (`select_agent_`), выбор действия (`action_`), или "Назад" (`back_to_agents`) |

### Под-ветка: выбор агента (select_agent_)

| Название | Назначение |
|---|---|
| **БД: Сохранить агента** | Сохраняет ID выбранного агента в `current_agent_id`. Проверяет через JOIN что агент принадлежит этому пользователю |
| **ТГ: Подтвердить выбор** | Подтверждает Telegram что кнопка обработана |
| **БД: Получить агента** | Загружает данные агента из БД включая `elevenlabs_api_key` пользователя |
| **EL: Данные агента** | GET-запрос в ElevenLabs — получает текущий промпт и приветствие. Использует API ключ из БД |
| **Код: Меню агента** | Формирует меню с текущими значениями промпта, приветствия, базы знаний и кнопками редактирования |
| **ТГ: Меню агента** | Отправляет меню агента в Telegram |

### Под-ветка: выбор действия (action_prompt / action_welcome / action_kb)

| Название | Назначение |
|---|---|
| **Код: Запрос ввода** | Разбирает callback-данные, определяет тип действия |
| **БД: Сохранить действие** | Сохраняет действие в `pending_action` — бот ждёт текстовый ввод |
| **ТГ: Подтвердить действие** | Подтверждает Telegram что кнопка обработана |
| **Код: Подсказка ввода** | Формирует текст подсказки что нужно ввести |
| **ТГ: Запросить ввод** | Отправляет подсказку пользователю |

### Под-ветка: кнопка "Назад к агентам"

| Название | Назначение |
|---|---|
| **БД: Сбросить pending при назад** | Очищает `pending_action` |
| **ТГ: Подтвердить назад** | Подтверждает Telegram что кнопка обработана |
| **БД: Получить агентов** | Общий нод — загружает список агентов |

---

## Ветка произвольного текста

| Название | Назначение |
|---|---|
| **БД: Получить состояние** | Загружает состояние пользователя: `pending_action`, выбранный агент, `elevenlabs_api_key` |
| **Проверка: Ожидает ключ** | Проверяет `pending_action = 'setkey'`. Если да — переходит к сохранению ключа |
| **БД: Сохранить ключ** | Сохраняет API ключ в `users.elevenlabs_api_key`, сбрасывает `pending_action` |
| **EL: Получить агентов (ключ)** | GET-запрос к ElevenLabs с новым ключом — проверяет ключ и получает агентов |
| **Код: Синх агентов (ключ)** | Формирует SQL для полной синхронизации агентов |
| **БД: Удалить старых (ключ)** | Удаляет устаревших агентов из БД |
| **БД: Upsert агентов (ключ)** | Добавляет/обновляет актуальных агентов |
| **ТГ: Ключ принят** | Отправляет "✅ API ключ успешно подключён!" |
| **Проверка: Агент выбран** | Проверяет: агент выбран И `pending_action` не пустой. Если нет — сообщение об ошибке |
| **Код: Парсинг LLM** | Определяет действие и значение на основе `pending_action` из БД и текста пользователя |
| **Роутер: Действие** | Направляет к нужному ElevenLabs-запросу: промпт / приветствие / база знаний |

---

## Обновление агента в ElevenLabs

Все ноды используют динамический `xi-api-key` заголовок из `users.elevenlabs_api_key`.

| Название | Назначение |
|---|---|
| **ElevenLabs: Промпт** | PATCH — обновляет системный промпт агента |
| **ElevenLabs: Приветствие** | PATCH — обновляет первое сообщение агента (first_message) |
| **ElevenLabs: База знаний** | POST — создаёт новый текстовый документ в базе знаний |
| **ElevenLabs: Привязать KB** | PATCH — привязывает созданный документ к агенту |

---

## Завершение обновления

| Название | Назначение |
|---|---|
| **БД: Сбросить действие** | Очищает `pending_action` после успешного обновления |
| **ТГ: Агент обновлён** | Отправляет "✅ Готово! Агент обновлён." |
| **EL: Данные после обновления** | GET — получает свежие данные агента для отображения |
| **Код: Меню после обновления** | Формирует меню с актуальными значениями |
| **ТГ: Меню после обновления** | Отправляет обновлённое меню агента |

---

## Сообщения об ошибках

| Название | Назначение |
|---|---|
| **ТГ: Нет действия** | "Сначала выбери агента" — агент не выбран или `pending_action` пустой |
| **ТГ: Нет ключа** | "Пришли свой ElevenLabs API ключ:" — ключ не привязан |

## Схема потока (кратко)

```
ТГ: Входящие → Роутер: Тип сообщения
  ├── /start → БД: Регистрация → Проверка: Ключ при старте
  │               ├── есть ключ → ТГ: Приветствие
  │               └── нет ключа → БД: Запросить ключ → ТГ: Нет ключа
  │
  ├── /agents → БД: Авторег /agents → БД: Получить ключ /agents → Проверка: Есть ключ
  │               ├── есть ключ → EL: Получить агентов /agents → Синх → БД: Получить агентов → Список
  │               └── нет ключа → БД: Запросить ключ → ТГ: Нет ключа
  │
  ├── callback → Роутер: Тип callback
  │     ├── select_agent → БД: Сохранить агента → ТГ: Подтвердить выбор
  │     │                → БД: Получить агента → EL: Данные агента → Код: Меню агента → ТГ: Меню агента
  │     ├── action_* → Код: Запрос ввода → БД: Сохранить действие → ТГ: Подтвердить действие
  │     │           → Код: Подсказка ввода → ТГ: Запросить ввод
  │     └── back → БД: Сбросить pending при назад → ТГ: Подтвердить назад → БД: Получить агентов → Список
  │
  └── текст → БД: Получить состояние → Проверка: Ожидает ключ
        ├── setkey → БД: Сохранить ключ → EL: Синх агентов → ТГ: Ключ принят → БД: Получить агентов → Список
        └── иное → Проверка: Агент выбран
              ├── Да → Код: Парсинг LLM → Роутер: Действие
              │     → ElevenLabs: [Промпт / Приветствие / (База знаний → Привязать KB)]
              │     → БД: Сбросить действие → ТГ: Агент обновлён
              │     → EL: Данные после обновления → Код: Меню после обновления → ТГ: Меню после обновления
              └── Нет → ТГ: Нет действия
```
